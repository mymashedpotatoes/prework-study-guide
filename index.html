HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • !DOCTYPE tells the browser what document type to expect when it is opened.
  • The title element defines title for the webpage.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as character set, description, keywords, author, and viewpoint.
  • The script element embeds client-side scripts in the HTML document.

CSS

  • Seperation of Concerns: Seperating code in different files based on use. "Each section of code has its own responsibilities.
  • Selector defines element and attributes to which the rules and declaration apply.
  • Margin indicates how much spare space we want around the OUTSIDE of an element. If 0 then NO extra space is around the element.
  • Padding adds space around content INSIDE an element. Increased padding means bigger element.
  • * applies rule to all elements visible on page.
  • , combines 2 selectors into single rule. So you don't have to repeat the rule twice. DRY
  • text-align: Centers title or heading on page.
  • img: Assigns property values to any image element in markup.
  • link: Creates the connection to the external file.
  • rel: Specifies the relationship between current document and the linked document resource.
  • href: Specifies the location, or URL, of the external resource. It is a relative path. Location is derived from the current location.
  • The period before card designates this as a class selector. THe following will gain these property assignments from the CSS.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • code .: to open VScode.
  • cd: navagates between directories.
  • pwd: displays current working directory.
  • git add -A: used to commit all files in the working branch. Add changed files from our working directory to staging area.
  • git commit -m "add message": to commit files to our repo, -m to associate message with out commit
  • git pull origin main: to see if current branch is up to date before pulling.
  • git push origin branch-name: push up all changes.
